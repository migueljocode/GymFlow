# 💪 GymFlow - سیستم مدیریت تمرین و ثبت وزن

**GymFlow** یک نرم‌افزار برنامه‌ریزی تمرین و ثبت وزن برای باشگاه‌های بدنسازی و مربیان شخصی است.  
کاربران می‌توانند برنامه تمرین هفتگی خود را بر اساس روزها و گروه‌های عضلانی تنظیم کرده، وزن خود را ثبت کنند و روند تغییرات وزن را همراه با پیش‌بینی آینده مشاهده نمایند.

---

## ✨ ویژگی‌های اصلی

- 📝 **برنامه تمرین هفتگی**  
  تعریف روزهای تمرین، گروه عضلانی (مثل روز پا، روز سینه، روز پشت) و توضیحات تمرین

- ⚖️ **ثبت و پیگیری وزن**  
  ثبت وزن در تاریخ‌های مختلف و نمایش روند تغییرات به صورت نمودار

- 🔮 **پیش‌بینی وزن**  
  محاسبه پیش‌بینی وزن آینده بر اساس میانگین تغییرات خطی (بدون نیاز به یادگیری ماشین)

- 📄 **خروجی PDF**  
  دریافت برنامه تمرین هفتگی به صورت فایل PDF قابل چاپ

- <i class="fas fa-user"></i> **حساب کاربری**  
  ثبت‌نام و ورود با ASP.NET Core Identity | هر کاربر داده‌های خود را جداگانه دارد

- 📊 **داشبورد شخصی**  
  مشاهده خلاصه فعالیت‌ها، آخرین وزن ثبت شده و برنامه تمرین جاری

---

## 🛠️ تکنولوژی‌های استفاده شده

| دسته | فناوری |
|------|--------|
| **فریم‌ورک اصلی** | ASP.NET Core 8 |
| **مدیریت دیتابیس** | Entity Framework Core + SQLite |
| **احراز هویت** | ASP.NET Core Identity |
| **فرانت‌اند** | Razor Pages + Bootstrap 5 |
| **مستندسازی API** | Swagger / OpenAPI |
| **تولید PDF** | QuestPDF |
| **نمودار** | Chart.js |

> **توجه:** دیتابیس SQLite فعلاً استفاده شده (بدون نیاز به نصب سرور جداگانه). در صورت نیاز به MySQL یا SQL Server، فقط کافی است Provider مربوطه در EF Core تغییر کند.

## 🚀 نحوه اجرا (روی سیستم شخصی)

### پیش‌نیازها
- [.NET 10 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/10.0)
- یک ویرایشگر مثل VS Code، Visual Studio یا JetBrains Rider

### مراحل اجرا

1. **کلون یا دانلود پروژه**
   ```bash
   git clone https://github.com/your-username/GymFlow.git
   cd GymFlow
   ```

2. **بازیابی پکیج‌ها**
   ```bash
   dotnet restore
   ```

3. **ساخت دیتابیس (اجرای Migration)**
   ```bash
   cd GymFlow.Dal
   dotnet ef migrations add InitialCreate
   dotnet ef database update
   cd ..
   ```

4. **اجرای همزمان API و Web**  
   از دستور زیر در ریشه پروژه استفاده کن:
   ```bash
   dotnet run --project GymFlow.Api & dotnet run --project GymFlow.Web
   ```
   
   یا در **Visual Studio**: پروژه‌های `GymFlow.Api` و `GymFlow.Web` را به عنوان Startup Projects تنظیم کن.

5. **باز کردن در مرورگر**
   - Web UI: `https://localhost:5001` یا `http://localhost:5000`
   - Swagger API Docs: `https://localhost:7001/swagger`
